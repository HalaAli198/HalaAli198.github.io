About

I am a PhD candidate in Computer Science at Virginia Commonwealth University, where I work in the Security and Forensics Engineering Lab. My research focuses on memory forensics, malware analysis, software supply chain security, runtime SBOM generation, and eBPF-based monitoring of malicious ML models.

My recent work develops memory- and runtime-based techniques for recovering software behavior after execution, including Volatility 3 plugins for Python and Go runtime analysis, runtime SBOM generation from Python memory, and system-call-based detection of malicious ML model behavior.

Current Research Projects

Memory Forensics of Go Malware

Developing Volatility 3 plugins to reconstruct runtime artifacts of Go-based malware, including function metadata, goroutines, channels, strings, interfaces, and heap objects.

MEM-SBOM: Memory-based Runtime SBOM Generation

Developing memory forensic techniques to generate runtime SBOMs for Python applications, recover dependency graphs, identify dynamically injected components, and analyze vulnerable-code reachability from memory.

ModelShield

Building an eBPF-based runtime monitoring framework for detecting malicious ML models by correlating application-level ML API calls with kernel-level syscall behavior.

3D Printing Security

Designing techniques for detecting malicious G-code manipulations and investigating illegal 3D printing activities using machine learning and memory forensics.
